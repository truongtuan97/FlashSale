# Order Idempotency Key Implementation

Created at: 2026-07-12 17:03 Asia/Bangkok

## Context

`POST /api/orders` now requires duplicate-submission protection.

Flash sale order creation immediately increases `FlashSaleItem.SoldQuantity`, so a retry from a browser, mobile client, reverse proxy, or unstable network must not create a second order or subtract stock twice.

## Decision

Require the `Idempotency-Key` header for:

```text
POST /api/orders
```

The API behavior is:

- Missing `Idempotency-Key` returns `400 Bad Request`.
- Key longer than 200 characters returns `400 Bad Request`.
- First request for a key creates the order and stores the key.
- Retrying the same key with the same request body returns the existing order.
- Reusing the same key with a different request body returns `409 Conflict`.

## Persistence

Added an order idempotency table through EF Core migration:

```text
20260712090600_AddOrderIdempotencyKeys
```

The table stores:

- Key
- RequestHash
- OrderId
- CreatedAt
- UpdatedAt

`Key` has a unique index.

The idempotency record and order are saved in the same database transaction.

## Request Hashing

The create-order request body is serialized with web JSON defaults and hashed with SHA-256.

The hash is used to distinguish:

- Same key + same request: safe retry.
- Same key + different request: conflict.

## Query Shape

Order response loading is centralized in `IncludeOrderResponseDetails`.

The response requires:

```text
Order
  -> Items
      -> FlashSaleItem
          -> Product
          -> Campaign
```

EF Core uses two include branches for `Product` and `Campaign` because they are sibling navigations under `FlashSaleItem`.

## Tests

Updated normal order integration tests to send `Idempotency-Key` through a `PostOrderAsync` helper.

Added idempotency coverage:

- Create order requires `Idempotency-Key`.
- Same key and same request returns existing order without increasing stock again.
- Same key and different request returns conflict.

Updated PostgreSQL order concurrency tests to also send `Idempotency-Key`.
