## Beginner Mental Model

The current backend layers should be understood like this:

```text
Domain         = the core business objects, for example Product
Infrastructure = database details, EF Core DbContext, table configuration
Api            = HTTP endpoints that the browser or frontend calls
Application    = use cases and business workflows, to be filled in later
Worker         = background jobs, to be filled in later
```

For the first working slice, the goal is:

```text
Browser/Swagger -> API endpoint -> FlashSaleDbContext -> PostgreSQL products table
```

Once that works, the project has its first real end-to-end backend feature.
